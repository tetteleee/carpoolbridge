# 配車アシスタント プロトタイプ

Version: 0.1

---

# 1. 目的

配車担当が実際に使えるUIかどうかを、コードを書く前に検証する。

本ドキュメントはFigma等の清書前段階として、
画面構成・情報配置・操作フローをテキストベースのワイヤーフレームで表現する。

検証方法:配車担当3人にこのドキュメント（または印刷したもの）を見せ、
「これなら使えるか」「入力・確認しやすいか」をヒアリングする。

---

# 2. アプリ名

**配車アシスタント**

---

# 3. 設計方針（配車担当の思考順に合わせる）

配車担当は概ね以下の順番で考える（00_現状業務分析・06_配車アルゴリズムと共通）。

```
集合場所ごとに分ける
↓
車を決める
↓
定員確認
↓
方向確認
↓
微調整
```

この思考順に沿って、画面上でも
「集合場所が色でひと目に分かる」→「車ごとにまとまって見える」→「定員・方向がすぐ確認できる」
という情報の出し方を優先する。

このアプリの最重要価値は、

> 「配車作成」ボタンを押して、80〜90%完成した配車案が10秒程度で出てくること

である。配車担当が一番困っているのは「定員オーバー」と「乗せ忘れ」なので、
この2つは常に画面上で目立たせる。

# 7. イベント編集（配車担当がLINEを見ながら入力）

参加・帰り・車出し・乗車可能人数・特記事項など、イベントに関する回答情報を
まとめて編集する画面。単なる「回答の入力」ではなく、後からいつでも
修正できる「編集」画面として位置づける。

家庭ごとにカードをまとめ、その中に所属する子供（1人〜複数人）の回答を並べる。
LINE返信自体が家庭単位で届くため、入力の単位も家庭に合わせる。

```
┌─────────────────────────┐
│ 7/12(土) 練習試合          │
├─────────────────────────┤
│ 山田家                    │
│                          │
│  車出し(行き) [可][不可]  │
│  車出し(帰り) [可][不可]  │
│  乗車可能人数 [__]人      │
│  ─────────────────      │
│  山田 太郎（小6）          │
│   参加 [○][✕][現地]       │
│   帰り [配車○][迎え][現地解散]│
│   □ 午後から参加          │
│   備考 [______________]  │
│  ─────────────────      │
│  山田 花（小3）            │
│   参加 [○][✕][現地]       │
│   帰り [配車○][迎え][現地解散]│
│   □ 午後から参加          │
│   備考 [______________]  │
├─────────────────────────┤
│ 佐藤家                    │
│                          │
│  車出し(行き) [可][不可]  │
│  車出し(帰り) [可][不可]  │
│  乗車可能人数 [__]人      │
│  ─────────────────      │
│  佐藤 花子（小4）          │
│   参加 [○][✕][現地]       │
│   帰り [配車○][迎え][現地解散]│
│   □ 午後から参加          │
│   備考 [______________]  │
│  ...                    │
├─────────────────────────┤
│ ← イベント一覧　配車画面へ →│
└─────────────────────────┘
```

- 家庭カードの上部に、その家庭に共通する情報（車出し可否・乗車可能人数）をまとめる
- カード内に子供ごとの回答（参加・帰り・午後参加・備考）を1人ずつ並べる
  （兄弟が複数人いる家庭でも、1枚のカードで完結する）
- 「午後から参加」「現地集合」はチェックボックス/ボタンで選択式にし、
  それ以外の例外は子供ごとの備考欄（自由入力）でカバーする
- 家庭カードは並び替えできるようにする。並び替え基準は以下の4種類を用意する
  - 五十音順
  - 集合場所順
  - 学年順
  - 未回答優先（未回答の家庭を上に表示）
  画面上部にドロップダウンまたはタブで並び替え基準を切り替えられるようにする
- 「保存」ボタンは置かない。入力した瞬間に自動保存する（10章の自動保存方針と統一）。
  画面下部には「← イベント一覧」「配車画面へ →」のナビゲーションのみ配置する

# 8. 配車画面（メイン）

## 8.1 初回表示（自動配車前）

```
┌─────────────────────────┐
│ 7/12(土) 配車             │
├─────────────────────────┤
│ 未配車: 18人               │
│                          │
│ 🟦 山田太郎 小6 西明石駅   │
│ 🟨 佐藤花子 小4 大久保駅   │
│ 🟩 田中一郎 小5 大久保駅   │
│ ...                      │
│                          │
│      [配車作成]           │
└─────────────────────────┘
```

- 起動直後は未配車の子供一覧のみを表示し、「配車作成」ボタンを押してもらう
- 子供カードの色は集合場所ごとに色分け（8.4参照）

## 8.2 自動配車後（メイン画面）

画面はスマホ幅に収めるため、「一覧（概要）」と「詳細（タップで展開）」の2段構成にする。

### 8.2.1 一覧（概要）— 横スクロールで全車を俯瞰

```
┌──────────┬──────────┬──────────┐
│ 栗本号     │ 大船号     │ 未配車     │
│ 4/5       │ 3/4       │ 2人       │
│ 西明石     │ 大久保     │           │
│  ↓        │  ↓        │           │
│ 大久保     │ 会場       │           │
│  ↓        │           │           │
│ 会場       │           │           │
├──────────┼──────────┼──────────┤
│🟦山田太郎  │🟨田中一郎  │🟩鈴木次郎  │
│🟦山田花    │🟨伊藤三郎  │🟥高橋四郎  │
│           │🟩佐藤花子  │           │
└──────────┴──────────┴──────────┘
      [+ 車を追加]   [戻す]
```

- 車ごとに「車名」「乗車率（4/5等）」「巡回順（経由地）」をカード上部にまとめて表示する。
  詳細ビューを開かなくても一覧だけで巡回順が分かるようにする
- その下に、乗車メンバーを「色＋名前」の
  簡易チップで並べる（学年・集合場所名などの詳細は省き、ドラッグ操作で
  識別できる最低限の情報にとどめる）
- 定員超過の場合はカード枠を赤くし、⚠マークを表示
- 子供チップをドラッグして別の車の枠へドロップすると、その場で車が入れ替わる
- 車カード（栗本号・大船号など）をタップすると8.2.2の詳細ビューが開く
- 操作の競合を避けるため、子供チップは「タップ＝AI判断理由を表示（8.3）」
  「長押し＋ドラッグ＝車の移動」とジェスチャーを使い分ける（要検証:
  配車担当が違和感なく使い分けられるか確認したい）

### 8.2.2 詳細（タップで展開）— 1台分をフル表示

```
┌─────────────────────────┐
│ 栗本号                    │
│ 4/5                      │
│                          │
│ 西公園                    │
│  ↓                       │
│ 中央公園                   │
│  ↓                       │
│ 会場                      │
├─────────────────────────┤
│ 🟦 山田 太郎               │
│    小6                   │
│    西公園                 │
├─────────────────────────┤
│ 🟦 山田 花                 │
│    小3                   │
│    西公園                 │
├─────────────────────────┤
│ 🟩 田中 一郎               │
│    小5                   │
│    中央公園                │
├─────────────────────────┤
│         [閉じる]          │
└─────────────────────────┘
```

- 車名・乗車率・経由地（巡回順）をカード上部にまとめて表示
- その下に、乗車する子供を1人ずつ区切り線で分けて縦に並べる
  （集合場所の色ドット・氏名・学年・集合場所名を1人分のブロックとして表示）
- タップした子供のブロックをさらにタップすると、8.3のAI判断理由がその場で開く
  （詳細ビューでは車間移動を行わないため、タップ操作のみでよい）
- ドラッグ&ドロップでの車間移動は、8.2.1の一覧表示（複数車が同時に見える状態）で行う。
  詳細ビュー（1台のみ表示）を開いている間は、他の車が画面上に無いため
  ドラッグ移動はできない仕様とする（この点は要検証:配車担当が実際に試して
  違和感がないか確認したい）

- 未配車カードは一覧（8.2.1）に通常の車カードと同じ形式で表示されるが、
  未配車の子供がいなくなれば自動的に消える（常に固定表示され続けるわけではない）
- 「+ 車を追加」ボタンで、車出し回答の有無に関わらず車を手動追加できる
  （応援号もこのボタンから追加する。特別な専用ボタンは用意しない）
- 「戻す」ボタンを一覧画面に常設し、直前の修正を1手戻せる（スマホ操作前提のため常設ボタン化）
- ドラッグ&ドロップでの子供の移動は8.2.1の一覧表示で行う。移動時の確認ダイアログ等は
  出さず、自由に修正できるようにする

## 8.3 子供カードの詳細・AIの判断理由

子供カードをタップすると、AIがその割り当てを選んだ理由を表示する。
その子に実際に適用された条件だけを表示する（該当しない項目は出さない）。

```
┌─────────────────────────┐
│ 山田 太郎                 │
│ 小6                      │
│ 西明石駅                  │
├─────────────────────────┤
│ 集合場所一致               │
│ 経路上にある               │
│ 定員に余裕あり             │
│ 兄弟を同乗                 │
└─────────────────────────┘
```

表示しうる理由の例（該当する場合のみ表示）

- 集合場所一致:通常の集合場所とその車の巡回経路が一致している
- 経路上にある:目的地までの方向・巡回順に沿っている（06_配車アルゴリズムの
  方位角クラスタリングの結果）
- 定員に余裕あり:その車の定員に対して詰め込みすぎていない
- 兄弟を同乗:兄弟が同じ車に割り当てられている（兄弟がいる子供のみ表示）
- ドライバーの子:その車のドライバーの子供である（該当する場合のみ表示）

複数の理由が同時に成立する場合はすべて列挙する。
06_配車アルゴリズムのどの判定ロジックが「理由」として文言化できるかは、
実装時に対応関係を精査する必要がある（デバッグ用途としても活用できるよう、
できるだけロジックの判定結果をそのまま理由として出す設計にする）。

## 8.4 色分けルール

- 集合場所ごとに色を割り当て、子供カードの名前の左に色付きの四角（🟦🟨🟩など）で表示する
- カード自体の背景色は変えない（車カードは全て白背景で統一し、情報の見やすさを優先）

## 8.5 配車作成（自動配車）ボタン

- ボタン名は「配車作成」
- 実行するたびに以下の確認ダイアログを表示する

```
現在の配車は失われます。
作り直しますか？

     [はい]   [いいえ]
```

- 何度でも再実行可能
- 定員に対して配車しきれない人数がいる場合は、配車可能なところまで実施し、
  残りは「未配車」として表示する（エラーにはしない）

# 9. 配車確定・LINE共有

## 9.1 確定前チェックリスト

配車確定・LINE共有ボタンを押す前に、自動チェックリストを表示する。

```
┌─────────────────────────┐
│ ✓ 未配車なし               │
│ ✓ 定員超過なし             │
│ ✓ 全18人配車済み           │
│                          │
│    [LINEにコピーする]     │
└─────────────────────────┘
```

## 9.2 LINE共有テキスト（出力例）

絵文字は使用せず、現行のLINE投稿より読みやすい形に整理する。

```
【7/12 練習試合】

■栗本号（西明石→大久保→会場）
・山田太郎
・佐藤花子

■大船号（大久保→会場）
・田中一郎
・伊藤三郎

--------------------------------
```

# 10. 保存・全般仕様

- 修正内容は最後に変更した状態を自動保存する（明示的な保存ボタンは不要）
- ダークモードに対応する（詳細な配色は05_画面設計で定義）
- 配車品質スコアのような数値表示は行わない（担当者の総合判断を尊重する）
- 画面下部固定ボタンの有無・配置は本プロトタイプ段階では未確定（TBD、05_画面設計で検討）

---

# 11. 検証したいポイント（配車担当3人へのヒアリング項目）

- 「配車作成」ボタンを押してから配車案が出るまでの流れは分かりやすいか
- 定員超過・未配車の表示は目に留まりやすいか
- ドラッグ&ドロップでの修正は直感的か（2タップ方式の方が良いか）
- AIの判断理由（集合場所一致・経路上にある・定員に余裕あり・兄弟を同乗など）は
  納得感があるか、情報が多すぎないか
- 車の手動追加（+ 車を追加）は自然に見つけられるか
- LINE共有テキストの見やすさは現行のLINE投稿と比べてどうか
