以下のルールでタスクファイルを作成してください。

■ 参照元
[対象の設計書ファイル名を指定。例: docs/04_データ設計.md の
「capacityToday」セクション]

■ タスクの粒度
- 1タスク = 1PR/1コミットで完結する範囲に収める
- 複数レイヤー（データ構造・UI・認証）にまたがる場合は
  レイヤーごとにタスクを分割する
- 目安: 1タスクの変更対象は1〜3ファイル程度

■ 各タスクファイルに必須で含める項目
1. 対象設計書とセクション名（ref: ファイル名#見出し）
2. このタスクのゴール（1〜2行）
3. 変更対象ファイル（想定でよい）
4. 実装範囲（やること）
5. 実装範囲外（やらないこと・触らないこと）
6. 受け入れ条件（完了判定基準）
7. 依存タスク（先に終わってないといけないタスクがあれば）

■ 禁止事項
- 設計書に書かれていない仕様を勝手に補完しない
- タスクの中で設計変更を提案する場合は
  「提案」として明示し、タスク本文とは分離する
- 既存タスクの内容を再定義しない（矛盾があれば指摘のみ）

■ 出力形式
tasks/ 配下に1タスク1ファイルで出力
ファイル名: <タスク名>.md
