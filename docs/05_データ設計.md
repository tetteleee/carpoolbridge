# 1. データ設計方針

シンプルなデータ構造を優先する。

- Firestoreを利用する
- 正規化しすぎない
- 入力しやすさを優先する
- 将来の拡張を考慮する

# 3. Family（家庭）

1家庭につき1レコード

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | String | ○ | ID |
| familyName | String | ○ | 〇〇家 |
| coachName | String \| null | | コーチ名。この家庭にコーチが紐づく場合のみ入力。空欄（null）はコーチなし |
| vehicleCapacity | Number | ○ | 車の総定員（運転者本人を含む。車を持たない場合は0） |
| pickupLocationId | String | ○ | 家庭の集合場所（家庭に属する子供・コーチは全員この集合場所から乗車する） |
| isActive | Boolean | ○ | 在籍中（falseで卒団・非表示扱い） |
| createdAt | Timestamp | ○ | |
| updatedAt | Timestamp | ○ | |

### coachName

この家庭の保護者がコーチを兼ねている場合のみ入力する。
イベントごとの参加有無はResponse.coachParticipatingで管理する。

### 運転者・家庭の表示名

保護者名の個別入力は行わない。車カード・LINE共有・乗車メンバー表示に
出る運転者名は、familyNameから「家」を除いた文字列（例：山田家→山田）を
そのまま使用する。コーチが運転する場合はcoachNameをそのまま表示する。

※本設計では「保護者」というデータは持たない。運転者・乗車者の種別は
「子供（Child）」または「コーチ（coachName）」のいずれかのみとする。

# 4. Child（子供）

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | String | ○ | |
| familyId | String | ○ | |
| name | String | ○ | |
| schoolEntryYear | Number | ○ | 小学校の入学年度（例：2026）。学年はこの値から自動計算する |
| isActive | Boolean | ○ | 在籍中（falseで卒団・非表示扱い） |
| createdAt | Timestamp | ○ | |
| updatedAt | Timestamp | ○ | |

- 家庭を無効化したら、その家庭の子供も自動で無効化する
- 子供の集合場所は個別に設定できない。所属家庭の`Family.pickupLocationId`を常に使用する（同一家庭内で集合場所が分かれるケースは許容しない）

# 5. PickupLocation（集合場所）

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | String | ○ | |
| name | String | ○ | |
| latitude | Number \| null | ○ | 緯度。未入力の場合はnull（緯度経度未設定） |
| longitude | Number \| null | ○ | 経度。未入力の場合はnull（緯度経度未設定） |

### 緯度経度の未設定について

登録時点では地図上の座標が分からない場合があるため、latitude・longitudeは手入力必須項目とはせず、未入力の場合はnullを保存する。
自動配車処理は開始前に全集合場所の緯度経度が設定済みであることを検証し、nullが1件でも存在する場合はエラーで中断する
（[07_配車アルゴリズム.md#2.1](07_配車アルゴリズム.md)）。

# 7. Event（イベント）

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | String | ○ | |
| name | String | ○ | 練習試合など |
| date | String | ○ | 開催日（"YYYY-MM-DD"形式） |
| destinationId | String | ○ | |
| createdAt | Timestamp | ○ | |
| updatedAt | Timestamp | ○ | |

### date

開催日（YYYY-MM-DD）。時刻は保持しない。

### 緯度経度について

Eventは緯度経度を保持しない。`destinationId`経由で`Destination`から取得する
（画面上も目的地選択のみで、緯度経度の個別入力は存在しないため、二重保持を避ける）。

# 8. Response（イベント回答）

イベント配下に保存する。

1家庭につき1レコード。**ResponseドキュメントIDをfamilyIdとする**
（`events/{eventId}/responses/{familyId}`）。ドキュメントが存在しない家庭は
「未回答」として扱う（statusフィールドは持たない。回答済みかどうかの判定は、
対象家庭のドキュメントが作成済みかどうかで行う）。

## 家庭情報

| 項目 | 型 | 説明 |
| --- | --- | --- |
| driverOutward | Boolean \| null | 行き車出し。未回答（未選択）ならnull |
| driverReturn | Boolean \| null | 帰り車出し。未回答（未選択）ならnull |
| capacityToday | Number \| null | 当日乗車可能人数（運転者本人を含む総定員）の上書き。通常通り（未変更）ならnull |
| coachParticipating | Boolean \| null | コーチが参加するか。`Family.coachName`が空の家庭ではnull（入力対象外） |
| remarks | String | 特記事項（子供個別の特殊ケースもここに集約する） |

### driverOutward・driverReturn

画面上は「未選択（未回答）」「可」「不可」の3状態を取り得るため、`Boolean | null`とする（null=未選択）。
自動保存（#11参照）により、車出しボタンをタップする前に同一家庭の他の項目（乗車可能人数・子供の参加など）が
先に保存された場合でも、未選択の車出しをfalse（不可）として確定させないためにnullを許容する。

### coachParticipating

`Family.coachName`が設定されている場合のみ入力対象。コーチが紐づかない家庭では常にnullとする。

`coachParticipating = true` の場合のみコーチを配車対象とする。
`driverOutward`/`driverReturn`がtrueなら運転者、falseなら乗客として扱う（null＝未回答の場合は配車対象外として扱う。詳細は配車アルゴリズム側で規定する）。

### capacityToday

当日の荷物等で車の総定員（`Family.vehicleCapacity`）から変動がある場合のみ数値を設定する。
未変更の場合はnullとする。

## 子供情報

children[]

```json
[
  {
    "childId": "xxx",
    "isParticipating": true,
    "noOutwardRide": false,
    "noReturnRide": false
  },
  {
    "childId": "yyy",
    "isParticipating": null,
    "noOutwardRide": true,
    "noReturnRide": true
  }
]
```

| 項目 | 型 | 説明 |
| --- | --- | --- |
| childId | String | 子供ID |
| isParticipating | Boolean \| null | イベントに参加するかどうか。未選択=null、参加=true、欠席=false |
| noOutwardRide | Boolean | 行きの配車が不要かどうか（現地集合、午後から参加など） |
| noReturnRide | Boolean | 帰りの配車が不要かどうか（保護者迎え、現地解散など） |

### isParticipating

イベント自体に参加するかどうかを表す。画面上は「○／✕／未選択」の3状態があるため、
`Boolean | null`とする（null=未選択）。`false`（欠席）の場合は配車対象外となる。

### noOutwardRide

行きの配車が不要な場合（現地集合など）に `true` とする。

### noReturnRide

帰りの配車が不要な場合（保護者迎えなど）に `true` とする。

### 子供個別の特記事項について

子供ごとの`remarks`は持たない。集合場所変更・同乗者ありなど特殊なケースは、
家庭情報側の`remarks`（家庭備考）にまとめて入力する（画面設計に準拠）。

# 9. Carpool（配車結果）

イベント配下に保存する。

車ごとに1レコード。

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | String | ○ | |
| direction | Enum | ○ | "OUTWARD"（行き）または "RETURN"（帰り） |
| driverFamilyId | String | ○ | 運転者の所属家庭ID |
| driverIsCoach | Boolean | ○ | 運転者がコーチかどうか。表示名生成時に`Family.coachName`（true）か家庭表示名（false、3章参照）かを判定するために使用 |
| capacity | Number | ○ | 運転者本人を含む総定員 |

### 車名・運転者名について

`carName`・`driverName`は保持しない。表示に必要な名称は毎回`Family`から生成する。

- 運転者名：`driverIsCoach = true` なら `Family.coachName`、`false` なら家庭表示名
  （`familyName`から「家」を除いた文字列。3章参照）
- 車名：家庭表示名から生成（例：山田 → 山田号）

## 巡回順（集合場所）

routeOrder[]

```json
["locationId-nishikoen", "locationId-chuokoen"]
```

集合場所IDを巡回順に保持する配列。

## 乗車メンバー

運転者は`driverFamilyId`/`driverIsCoach`で表現する。membersには運転者を含めない。

種別はマスタに登録される`child`・`coach`の2種のみとする。
`child`は`childId`、`coach`は`familyId`＋`Family.coachName`で表現する
（保護者というデータ種別は存在しない）。

```json
[
  { "type": "child", "childId": "xxx" },
  { "type": "coach", "familyId": "family-002" }
]
```

### type: "child" について

`direction = OUTWARD`の場合は`Response.children[].isParticipating`が`true`かつ
`noOutwardRide`が`false`の子供が対象。`direction = RETURN`の場合は同様に
`isParticipating`が`true`かつ`noReturnRide`が`false`の子供が対象。

### type: "coach" について

`coachParticipating: true`かつ、対象方向（`driverOutward`/`driverReturn`）の
車出しが不可の場合のみmembersに含める。車出しが可でコーチが運転する場合は
`driverFamilyId`/`driverIsCoach`側で表現する。

# 11. 削除方針

物理削除は行わず、論理削除（`isActive`）を使用する。

家庭・子供が卒団した場合は、`isActive`を`false`にして非表示とする。

- 新規イベントの回答画面・配車画面には表示しない
- 過去のイベント履歴（Response・Carpool）は引き続き参照・表示できる
- 家庭・子供・イベント・回答・配車結果は物理削除しない

## 例外：開発用「サンプルデータ投入」機能

マスタ管理画面の開発用機能「サンプルデータ投入」（開発環境限定。本番環境では利用不可）に限り、
上記の削除方針の例外として、家庭・子供を物理削除してよい。

- 対象は集合場所・目的地・家庭・子供のマスタデータのみ
- イベント・回答・配車結果は本機能の対象外であり、削除しない
- 本機能は開発環境での評価・動作確認を目的とした一括リセット用であり、
  過去のイベント履歴が家庭・子供を参照する整合性は保証しない

## 例外：配車再作成

イベント編集 回答入力画面（04_画面設計.md#7）の「配車作成」から配車を再作成する場合に限り、
上記の削除方針の例外として、対象イベント配下の配車結果（Carpool）を物理削除してよい。

- 再作成確認ダイアログ（04_画面設計.md#7 配車再作成）で「再作成」が選択された場合のみ実行する
- 削除対象は対象イベント配下の配車結果（行き・帰り両方向）のみとし、家庭・子供・イベント・回答は削除しない
- 削除後、最新の回答内容を基に配車を再作成する（再作成前の配車結果の履歴は保持しない）
