# 1. データ設計方針

MVPではシンプルなデータ構造を優先する。

- Firestoreを利用する
- 正規化しすぎない
- 入力しやすさを優先する
- 将来の拡張を考慮する

# 3. Family（家庭）

1家庭につき1レコード

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | String | ○ | ID |
| familyName | String | ○ | 〇〇家 |
| coachName | String \| null | | コーチ名。この家庭にコーチが紐づく場合のみ入力。空欄（null）はコーチなし |
| vehicleCapacity | Number | ○ | 車の総定員（運転者本人を含む。車を持たない場合は0） |
| pickupLocationId | String | ○ | 家庭の基本集合場所（子供の`pickupLocationOverride`がnullの場合に使用） |
| isActive | Boolean | ○ | 在籍中（falseで卒団・非表示扱い） |
| createdAt | Timestamp | ○ | |
| updatedAt | Timestamp | ○ | |

### coachName

この家庭の保護者がコーチを兼ねている場合のみ入力する。
イベントごとの参加有無はResponse.coachParticipatingで管理する。

### 運転者・保護者の表示名

保護者名の個別入力は行わない。車カード・LINE共有・乗車メンバー表示に
出る運転者名は、familyNameから「家」を除いた文字列（例：山田家→山田）を
そのまま使用する。コーチが運転する場合はcoachNameをそのまま表示する。

# 8. Response（イベント回答）

イベント配下に保存する。

1家庭につき1レコード。

## 家庭情報

| 項目 | 型 | 説明 |
| --- | --- | --- |
| familyId | String | |
| status | Enum | "未回答" または "回答済み" |
| driverOutward | Boolean | 行き車出し |
| driverReturn | Boolean | 帰り車出し |
| capacityToday | Number \| null | 当日乗車可能人数（運転者本人を含む総定員）の上書き。通常通り（未変更）ならnull |
| coachParticipating | Boolean \| null | コーチが参加するか。`Family.coachName`が空の家庭ではnull（入力対象外） |
| remarks | String | 特記事項 |

### coachParticipating

`Family.coachName`が設定されている場合のみ入力対象。コーチが紐づかない家庭では常にnullとする。

`coachParticipating = true` の場合のみコーチを配車対象とする。
`driverOutward`/`driverReturn`がtrueなら運転者、falseなら乗客として扱う。

### status

"未回答" または "回答済み"。イベント作成直後の初期状態は "未回答"。
未回答の家庭は配車対象外。

### capacityToday

当日の荷物等で車の総定員（`Family.vehicleCapacity`）から変動がある場合のみ数値を設定する。
未変更の場合はnullとする。

## 子供情報

children[]

```json
[
  {
    "childId": "xxx",
    "isParticipating": true,
    "noOutwardRide": false,
    "noReturnRide": false,
    "remarks": ""
  },
  {
    "childId": "yyy",
    "isParticipating": true,
    "noOutwardRide": true,
    "noReturnRide": true,
    "remarks": "現地集合、帰りも別"
  }
]
```

| 項目 | 型 | 説明 |
| --- | --- | --- |
| childId | String | 子供ID |
| isParticipating | Boolean | イベントに参加するかどうか |
| noOutwardRide | Boolean | 行きの配車が不要かどうか（現地集合、午後から参加など） |
| noReturnRide | Boolean | 帰りの配車が不要かどうか（保護者迎え、現地解散など） |
| remarks | String | 特記事項（集合場所変更、同乗者ありなど） |

### isParticipating

イベント自体に参加するかどうかを表す。`false`（欠席）の場合は配車対象外となる。

### noOutwardRide

行きの配車が不要な場合（現地集合など）に `true` とする。

### noReturnRide

帰りの配車が不要な場合（保護者迎えなど）に `true` とする。

# 9. Carpool（配車結果）

イベント配下に保存する。

車ごとに1レコード。

| 項目 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | String | ○ | |
| driverFamilyId | String | ○ | 運転者が家庭の保護者・コーチいずれの場合も、その運転者の所属家庭IDを設定する |
| driverName | String | ○ | 保護者が運転する場合は3章末尾の表示名ルールで自動生成、コーチが運転する場合は`Family.coachName`をそのまま使用 |
| carName | String | | |
| capacity | Number | ○ | 運転者本人を含む総定員 |

## 巡回順（集合場所）

routeOrder[]

```json
["locationId-nishikoen", "locationId-chuokoen"]
```

集合場所IDを巡回順に保持する配列。

## 乗車メンバー

運転者はdriverFamilyId/driverNameで表現する。membersには運転者を含めない。

種別はマスタに登録される`child`・`coach`の2種のみとする。
`child`は`childId`、`coach`は`familyId`＋`name`（`Family.coachName`）で表現する。

```json
[
  { "type": "child", "childId": "xxx" },
  { "type": "coach", "familyId": "family-002", "name": "田中コーチ" }
]
```

### type: "child" について

`Response.children[].isParticipating`が`true`かつ`noOutwardRide`/`noReturnRide`が
`false`の子供が対象。

### type: "coach" について

`coachParticipating: true`かつ家庭の車出しが不可の場合のみmembersに含める。
車出しが可でコーチが運転する場合は`driverFamilyId`/`driverName`側で表現する。

# 11. 削除方針

物理削除は行わず、論理削除（`isActive`）を使用する。

家庭・子供が卒団した場合は、`isActive`を`false`にして非表示とする。

- 新規イベントの回答画面・配車画面には表示しない
- 過去のイベント履歴（Response・Carpool）は引き続き参照・表示できる
- 家庭・子供・イベント・回答・配車結果は物理削除しない
