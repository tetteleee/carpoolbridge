# Task008a 配車修正（ドライバー変更・巡回順変更・その他の乗車者追加）

Version: 1.2

---

# 目的

UC-06「配車を修正する」のうち、Task008（ドラッグ＆ドロップ）で
スコープ外とした以下3つの操作を実装する。

- ドライバー変更
- 巡回順（経由地の順序）変更
- 「その他の乗車者」（マスタに存在しない例外的な同乗者）の
  自由入力での手動追加・削除

---

# 背景

Task007（配車画面）・Task008（ドラッグ＆ドロップ）で、配車結果の表示・
子供やコーチのドラッグによる車間移動・複数手戻せるUndoまでは
実装済みという前提で本Taskを進める。

これら3操作は05_画面設計.md 8.2.2（詳細ビュー）に明確なワイヤーフレームが
無いため、本Taskで画面仕様を確定させる。

なお、当初検討していた「単独コーチ（家庭に紐づかないコーチ）」の
マスタ登録・選択方式は、現時点のチームに該当者がいないため見送り、
09_今後のアイデア.mdへ移した。代わりに、マスタ登録を必要としない
より汎用的な「その他の乗車者」追加機能（04_データ設計.md 10章の
`type: "other"`）で対応する。

---

# 前提とする決定事項

- 3操作はすべて詳細ビュー（8.2.2）に置く
- ドライバー変更：ドロップダウンで在籍中の家庭（`isActive: true`）から
  選択する。変更すると、その家庭の`capacityToday`（設定されていれば）
  または`Family.vehicleCapacity`（運転者本人を含む総定員）を新しい
  定員として自動反映する。定員自体の手動編集フィールドは本Taskでは
  設けない（定員変更はTask004の回答入力画面で行う既存フローに委ねる）
- ドライバー変更時、選択した家庭に当該イベントで`coachParticipating:
  true`のコーチが紐づいている場合はそのコーチを、いなければ
  `Family.parentName`を、それぞれ`isDriver: true`のメンバーとして
  Carpool.membersへ設定する（Task007のスタブロジックの運転者自動生成
  ルールと同じ判定を再利用する）
- 巡回順変更：ドラッグではなく上下ボタン（↑↓）で並び替える
  （経由地は通常2〜4件程度のため、Task008のドラッグ基盤を
  流用せず単純なUIにする）
- その他の乗車者の追加：「+ 人を追加」ボタンから、名前を自由入力して
  追加する（マスタ検索・登録は不要）。追加された乗車者には個別の
  「削除」ボタンを表示する（未配車リストはResponse由来の参加者専用の
  ため、その他の乗車者はそこには表示しない）
- 3操作は全てTask008の履歴スタックに積み、「戻す」の対象に含める
  （別のUndo系統は作らない）
- ドライバー変更により「元のドライバーの子供が別の車に取り残される」
  状態になった場合、Task008の警告システムに新しい警告種別
  「ドライバーの子が同乗していない」を追加して表示する
- 定員判定は`members.length > capacity`のみで行う（Task007・008の
  運転者統一設計により、`isDriver`の有無による特別な+1調整は不要）

# 完了条件

- 詳細ビューでドライバー名をタップすると、在籍中の家庭一覧から
  新しいドライバーを選択できる
- ドライバーを変更すると、旧ドライバーのメンバー（`isDriver: true`）が
  Carpool.membersから取り除かれ、新しいドライバーのメンバーが
  追加される（家庭にコーチが紐づいていればcoach、いなければparent）
- ドライバー変更に伴い、Carpoolの`driverFamilyId`・`driverName`・
  `capacity`が更新され、Firestoreへ即時反映される
- ドライバー変更により、元のドライバーの子供が同じ車にいなくなった
  場合、警告「ドライバーの子が同乗していない」が表示される
- 詳細ビューの経由地（巡回順）に↑↓ボタンが表示され、押すたびに
  その経由地の順序が入れ替わる
- 巡回順の変更がCarpoolの`routeOrder`に反映され、Firestoreへ
  即時反映される
- 詳細ビューに「+ 人を追加」ボタンがあり、押すと名前を自由入力して
  乗車者を追加できる
- 追加された乗車者は`type: "other"`のCarpoolMemberとして
  Firestoreへ保存される
- 手動追加された乗車者の行に「削除」ボタンがあり、押すと
  Carpool.membersから取り除かれる
- 上記3操作はいずれもTask008の履歴スタックに積まれ、「戻す」ボタンで
  取り消せる
- npm run build が成功する

# 実装内容

## 1. Firestoreアクセス関数の追加

`src/firestore/carpools.ts`（Task007で作成済み）に以下を追加する。

```typescript
// ドライバー変更・巡回順変更・その他の乗車者追加削除など、
// 1台分の部分更新に使う
export async function updateCarpool(
  eventId: string,
  carpoolId: string,
  input: Partial<Carpool>
): Promise<void> { ... }
```

---

## 2. ドライバー変更UI

詳細ビュー（8.2.2、Task007で実装済み）の車名・ドライバー表示部分を
タップ可能にし、タップすると家庭選択ドロップダウン（またはシート）を
表示する。

```
┌─────────────────────────┐
│ 栗本号                    │
│ ドライバー：山田太郎の親 [変更] │
│ 4/5                      │
│ ...
```

選択すると

1. 新しいドライバーの家庭情報（`vehicleCapacity`・当該イベントの
   `Response.capacityToday`があればそれ）を取得する
2. 当該イベントで`coachParticipating: true`のコーチが選択した家庭に
   紐づいているか確認する
3. Carpool.membersから、旧ドライバーのメンバー（`isDriver: true`）を
   取り除く
4. 新しいドライバーのメンバーを追加する。2で該当コーチがいれば
   `{ type: "coach", id, isDriver: true }`、いなければ
   `{ type: "parent", familyId, name: parentName, isDriver: true }`
5. Carpoolの`driverFamilyId`・`driverName`・`capacity`を更新する
6. 更新前の状態をTask008の履歴スタックへpushする
7. `updateCarpool`でFirestoreへ保存する
8. 5章の警告判定を再計算する

---

## 3. 巡回順変更UI

詳細ビュー上部の経由地表示に↑↓ボタンを追加する。

```
西公園 [↓]
  ↓
中央公園 [↑][↓]
  ↓
会場（固定、ボタンなし）
```

- 目的地（会場）は`routeOrder`の対象外とし、常に最後尾固定とする
- ↑↓ボタンで`routeOrder`内の該当要素を隣接要素と入れ替える
- 変更のたびに履歴スタックへpushし、`updateCarpool`で保存する

---

## 4. その他の乗車者の手動追加・削除UI

詳細ビューの乗車メンバー一覧の末尾に「+ 人を追加」ボタンを表示する。

```
┌─────────────────────────┐
│ ...（既存の乗車メンバー）
├─────────────────────────┤
│      [+ 人を追加]         │
└─────────────────────────┘
```

タップすると名前の自由入力欄を表示する。入力して確定すると

1. Carpool.membersへ`{ type: "other", name: 入力された名前 }`を
   追加する
2. 履歴スタックへpushする
3. `updateCarpool`でFirestoreへ保存する

手動追加された乗車者の行には「削除」ボタンを表示し、押すと
Carpool.membersから取り除く（履歴スタックへのpush・Firestore保存も
同様に行う）。

※ 自動配車由来のメンバー（子供・運転者・家庭紐づきコーチ・
いつもは参加しない保護者等）には「削除」ボタンは表示しない。
それらを取り除きたい場合はTask008のドラッグで未配車エリアへ
移動する操作を使う（既存仕様のまま。ただし運転者の削除・変更は
本Task2章「ドライバー変更」で行う）。

---

## 5. 警告種別の追加

`src/dispatch/warnings.ts`（Task008で作成済み）に以下を追加する。

```typescript
export type CarpoolWarning =
  | "定員超過"
  | "兄弟分離"
  | "集合場所分割"
  | "ドライバーの子が同乗していない";
```

判定：Carpoolの`driverFamilyId`が設定されている場合、その家庭の
子供のうち、このイベントに参加する子供（`isParticipating: true`）が
1人も`members`に含まれていなければ警告を表示する。
（`driverFamilyId`が空、つまり応援号が運転する車には
この警告を適用しない）

  一括編集・一括確定
