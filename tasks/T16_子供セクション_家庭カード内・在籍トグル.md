## 4. 実装範囲（やること）

### 子供セクションの表示

家庭カード内に「子供」セクションを配置する。

画面イメージ（04_画面設計.md#10より）

```
│  子供                    │
│    名前       [________] │
│    入学年度   ▼2025年度(小2) │
│    集合場所   ▼基本集合場所 │
│    在籍中     [ON/OFF]    │
│                  [+ 子供を追加] │
```

- 対象家庭に紐づく子供を一覧表示する（1人〜複数人）
- 各子供の以下の項目を表示・編集できるようにする
  - 名前
  - 入学年度（`schoolEntryYear`）。選択時に学年表示を併記する（例：2025年度(小2)）
  - 集合場所（`pickupLocationOverride`）。選択肢は「基本集合場所」＋登録済みの集合場所一覧とし、
    「基本集合場所」選択時は`pickupLocationOverride`をnullとして扱う
  - 在籍中トグル（ON/OFF）
- 「+ 子供を追加」ボタンで対象家庭に子供を追加できる

### 保存

- 追加・編集内容は家庭セクション（T15）と同様の保存操作でFirestoreへ反映する

## 5. 実装範囲外（やらないこと）

以下は対象外。

- 家庭カード自体の基本情報表示・編集（T15で実装済み想定）
- Child CRUD処理そのものの実装（T12で実装済みのService層を利用する）
- 家庭無効化時の子供自動連動無効化ロジック（T12側の責務。本タスクは
  子供個別の在籍中トグル操作のみを対象とする）
- 学年計算ロジックそのものの実装（`schoolEntryYear`からの学年算出処理は
  別途提供されるユーティリティを利用する想定とし、本タスクでは表示への組み込みのみ行う）
- 削除操作（T15と同様、本画面は在籍中トグルによる論理無効化のみを対象とし、
  物理削除UIは対象外とする）

これらは別タスクまたは対象外とする。

---

## 6. 受け入れ条件

- 家庭カード内に「子供」セクションが表示される
- 対象家庭に紐づく子供が一覧表示される
- 「+ 子供を追加」で対象家庭に子供を追加できる
- 名前・入学年度・集合場所・在籍中トグルを編集できる
- 入学年度選択時に学年表示が併記される
- 集合場所は「基本集合場所」または登録済みの集合場所から選択できる
- 在籍中トグルのON/OFFが切り替えられる
- 保存操作でFirestoreへ反映される
