# Task003 イベント作成

Version: 1.0

---

# 目的

05_画面設計 6章のイベント作成画面を実装する。

Firebase/Firestoreとの連携はTask005（Firebase認証・権限）以降で行うため、
本TaskではモックのインメモリストアでイベントをCRUDし、
画面のみを完成させる。

---

# 完了条件

- `/events/new` でイベント作成フォームが表示される
- イベント名（テキスト）・日付・目的地（ドロップダウン）を入力できる
- 「保存」を押すとイベントが追加され、ホーム画面（`/`）へ遷移する
- 保存したイベントがホーム画面の一覧に表示される
- バリデーション：イベント名・日付は必須。未入力時は保存ボタンを無効にする
- npm run build が成功する

# 実装内容

## 1. ルーティングの追加

`src/App.tsx` に `/events/new` ルートを追加する。

```tsx
<Route path="/events/new" element={<CreateEventPage />} />
```

既存のルート（`/`・`/event/:id`・`/event/:id/dispatch`・`/master`）は変更しない。

---

## 2. モックストアの作成

`src/mocks/eventStore.ts` を新規作成する。

```typescript
import type { Event } from "../types";
import { mockEvents } from "./events";

// モジュールスコープでイベント一覧を保持する（インメモリストア）
let events: Event[] = [...mockEvents];

export function getEvents(): Event[] {
  return events;
}

export function addEvent(event: Event): void {
  events = [...events, event];
}
```

---

## 3. Event型の拡張

`src/types.ts` の `Event` に `date` フィールドはすでに存在する。
本Taskで追加が必要なフィールドはない。

ただし、目的地ドロップダウン用の `Destination` 型を追加する。

```typescript
export interface Destination {
  id: string;
  name: string;
}
```

## 4. モック目的地データ

`src/mocks/destinations.ts` を新規作成する。

```typescript
import type { Destination } from "../types";

export const mockDestinations: Destination[] = [
  { id: "d1", name: "○○小学校" },
  { id: "d2", name: "△△グラウンド" },
  { id: "d3", name: "△△球場" },

```

---

## 5. CreateEventPage の作成

`src/pages/CreateEventPage.tsx` を新規作成する。

表示内容（05_画面設計 6章のワイヤーフレームに準拠）

```
イベント作成
──────────────────
イベント名
[__________________]

日付
[2026/07/12]

目的地
▼ ○○小学校（履歴から選択）

         [保存]
```

- イベント名：テキスト入力（必須）
- 日付：`<input type="date">`（必須、初期値は今日の日付）
- 目的地：`mockDestinations` からドロップダウン選択（任意）
  - 未選択時は `destinationName` を空文字で保存
- 保存ボタン：イベント名・日付が入力済みの場合のみ活性
- 保存処理：`addEvent` を呼び出してホーム（`/`）へ遷移する
- 戻るリンク：画面上部に「← イベント一覧」を配置し `useNavigate` で戻れるようにする

---

## 6. HomePage のストア参照への切り替え

`src/pages/HomePage.tsx` が `mockEvents` を直接参照している箇所を
`getEvents()` に切り替える。

```typescript
// 変更前
import { mockEvents } from "../mocks/events";
const sortedEvents: Event[] = [...mockEvents].sort(...);

// 変更後
import { getEvents } from "../mocks/eventStore";
// コンポーネント内で呼び出す（毎回最新を取得するため）
const sortedEvents = [...getEvents()].sort((a, b) => a.date.localeCompare(b.date));
```

※ モックストアはインメモリのため、ページリロードでリセットされる。
Task005のFirestore連携以降でリロード耐性を持たせる。

---

## 7. IDの採番

新規イベントのIDは `crypto.randomUUID()` を使用する。

```typescript
id: crypto.randomUUID(),
```
