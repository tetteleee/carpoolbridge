# Task T17b Playwright導入・E2E基盤構築

---

## 1. 対象設計書

ref: なし（開発基盤に関するタスクのため、対象の設計書セクションはなし）

---

## 2. このタスクのゴール

PlaywrightによるE2Eテストの実行基盤を導入する。以降の画面実装タスクでは、T17aで到達可能になったマスタ管理画面をはじめ、実装済みの画面に対してE2Eテストを追加しながら開発を進められる状態にする。

---

## 3. 変更対象ファイル（想定）

- `package.json`
- `playwright.config.ts`（新規）
- `e2e/`（新規ディレクトリ）
- `.github/workflows/`（E2E実行用ワークフロー、新規または既存ワークフローへの追記）

※ファイル構成はプロジェクト構成に合わせて読み替えてよい。

---

## 4. 実装範囲（やること）

- Playwrightを導入する
- `playwright.config.ts`を作成する
- GitHub ActionsにE2E実行用のワークフローを追加する
- 最低限のE2Eテストを1件作成する（例: マスタ管理画面のURLに直接アクセスできることを確認するテスト）

---

## 5. 実装範囲外（やらないこと）

- マスタ管理画面以降の各画面に対する網羅的なE2Eテストの作成（各画面の実装タスク・ルーティング追加タスクの中で個別に追加する）
- 単体テスト・結合テストの導入（対象外。E2Eのみ）

---

## 6. 受け入れ条件

- `npx playwright test`相当のコマンドでE2Eテストが実行できる
- GitHub Actions上でE2Eテストが実行される
- マスタ管理画面のURLに直接アクセスできることを確認するテストが1件以上存在し、成功する

---

## 7. 依存タスク

- T17a マスタ管理画面 ルーティング追加
