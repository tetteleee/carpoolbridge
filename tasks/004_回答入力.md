# 004_回答入力（イベント編集）の実装

## 目的
LINEで収集した各家庭からの出欠や車出しなどの回答を、イベントに対して入力（編集）する画面を実装する。

## 実装内容
### 1. 型定義とモックデータの追加
- `src/types.ts` に以下の型を追加する（04_データ設計.md 参照）
  - `Family`（id, name, vehicleCapacity 等）
  - `Child`（id, familyId, name, grade, pickupLocationId 等）
  - `Response`（イベントごとの回答: familyId, status, driverOutward, driverReturn, capacityToday, remarks, children[]）
  - `ChildResponse`（isParticipating, noOutwardRide, noReturnRide, remarks）
- `src/mocks/families.ts`（または適宜）に家庭・子どものモックデータを作成する（3〜5家族分程度）。

### 2. 回答情報のステート管理（MVP用）
- 現在メモリ内でイベントを管理している `eventStore.ts` （または専用の `responseStore.ts`）に、回答データ（`Response[]`）を保存・取得・更新できる処理を追加する。
- 編集画面を開いた際に、まだ回答データが存在しない家庭については、初期状態（status: "未回答"）の `Response` データを自動生成する。

### 3. イベント編集画面（`EventEditPage.tsx`）の実装
- **画面ヘッダー**: 
  - 対象のイベント名・日付を表示。
  - 左上に「← イベント一覧」リンク。
  - 右上に「自動配車へ →」ボタン（Task 005用）。
- **家庭カードのリスト**:
  - マスターに登録されている全家庭をカード状に並べて表示する。
- **家庭カード（ヘッダー部分）**:
  - 家族名
  - **回答状況トグル**: `[未回答 / 回答済]`
  - **車出し(行き) / (帰り)**: チェックボックスまたはトグル
  - **乗車可能人数**: 
    - 未変更（`capacityToday: null`）の場合は、マスターの `Family.vehicleCapacity` を薄字でプレースホルダー表示する。
    - 値を変更した場合は数値を上書き保存し、画面上では `⚠️` アイコン等で「変更済み」であることが分かるように強調表示する。
- **家庭カード（子ども部分）**:
  - その家庭に属する子どもをリスト表示。
  - **参加**: `[○ / ✕]` ボタンまたはトグル
  - **行きの配車不要**: チェックボックス
  - **帰りの配車不要**: チェックボックス
  - **備考**: テキスト入力
- **親切UI（自動回答済フラグ）**: 
  - 初期状態は「未回答」だが、配車担当が車出しや子どもの項目を何か操作した瞬間に、自動的にその家庭のステータスを「回答済」に切り替える。

## 完了条件
- ホーム画面から対象イベントの「回答入力」画面を開けること。
- 全家庭のカードが表示され、子供ごとの出欠や乗車人数の上書き操作ができること。
- 「未回答」の家庭の項目を編集すると、自動的に「回答済」ステータスになること。
- 画面を離れて再度開いた際に、編集した回答状態や上書きした人数が正しく復元（保持）されること。
