# Task004a 追加の同乗保護者・定員表記の更新

Version: 1.0

---

# 目的

回答入力画面（Task004、Task007aでFirestore化済み）に、
「いつもは参加しない保護者等が今回だけ同乗する」場合の入力欄を追加する。

あわせて、「乗車可能人数」が運転者本人を含む総定員を表すことを
画面上でも明確にする（04_データ設計.md 3・9章の定義変更に伴う表記統一）。

---

# 背景

配車の定員を「運転者を含む総定員」に統一する設計変更に伴い、
04_データ設計.md 9章に`Response.additionalParentNames`（家庭単位で
複数人登録できる、自由入力の同乗者名リスト）が追加された。

運転者そのもの（車を出す本人）は、既存の車出し（行き／帰り）の
回答から自動的に配車結果へ反映されるため、新しい入力は不要。
本Taskで追加するのは、あくまで運転以外で同乗する人物のみである。

---

# 完了条件

- 家庭カード内に「追加の同乗者」欄があり、名前を自由入力で
  複数人追加できる
- 追加した同乗者は`Response.additionalParentNames`（文字列配列）として
  Firestoreへ保存される
- 各同乗者に削除ボタンがあり、個別に削除できる
- 「乗車可能人数」の項目名またはヘルプテキストに、運転者本人を
  含む総定員であることが分かる説明が付与されている
- 画面を離れて再度開いた際に、追加した同乗者が正しく復元される
- npm run build が成功する

---

# 実装内容

## 1. 型定義の更新

`src/types.ts`の`Response`型に`additionalParentNames: string[]`を
追加する（04_データ設計.md 9章準拠）。

```typescript
export interface Response {
  // ...既存フィールド
  additionalParentNames: string[];
}
```

新規生成される初期Response（`getOrCreateResponse`、Task007aで実装済み）
のデフォルト値は空配列`[]`とする。

---

## 2. 家庭カードUI

家庭カード内、乗車可能人数の下あたりに以下を追加する。

```
─────────────────
追加の同乗者（いつもは参加しない人）
 [__________________] [削除]
 [__________________] [削除]
      [+ 同乗者を追加]
```

- 「+ 同乗者を追加」を押すと、空のテキスト入力欄が1行追加される
- 各行の「削除」ボタンで、その行を`additionalParentNames`から取り除く
- テキスト入力の変更は、既存の自動保存パターン（Task007aで実装済みの
  `upsertResponse`呼び出し）に合わせて即時Firestoreへ反映する
- 空文字列の行は保存時に除外する（トリムして空になった行も同様）

---

## 3. 乗車可能人数の表記更新

既存の「乗車可能人数」ラベルまたはその近くに、
「（運転者本人を含む）」等の補足テキストを追加する。

```
乗車可能人数（運転者本人を含む） [ 5 ]人
```

既存のプレースホルダー表示・変更済みアイコンの挙動
（Task004実装済み）は変更しない。

# スコープ外

以下は実装しない。

- 追加の同乗者に対する集合場所・往復別の個別設定
  （所属家庭の子供と同じ集合場所を自動的に使う前提のため、
  04_データ設計.md 9章参照）
- 追加の同乗者のマスタ登録・個人ID管理
  （自由入力の名前のみを保持する）
- 配車画面側の反映ロジック（Task007のスタブロジック側で対応）
