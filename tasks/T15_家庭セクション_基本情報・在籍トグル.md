## 4. 実装範囲（やること）

### 家庭セクションの表示

マスタ管理画面内に「家庭」セクションを配置する。

画面イメージ（04_画面設計.md#10より）

```
├─────────────────────────┤
│ 家庭                    │
│  家庭名     [__________] │
│  コーチ     [__________] │
│  通常定員   [__]         │
│  基本集合場所 ▼○○公園     │
│  在籍中     [ON/OFF]      │
│                         │
├─────────────────────────┤
│ 家庭                    │
│        ...              │
├─────────────────────────┤
│                  [+ 家庭を追加] │
```

- 登録済みの家庭を一覧表示する（家庭カード単位）
- 各家庭カードに以下の項目を表示・編集できるようにする
  - 家庭名
  - コーチ名（空欄可。空欄＝コーチなし）
  - 通常定員（`vehicleCapacity`）
  - 基本集合場所（登録済みの集合場所から選択）
  - 在籍中トグル（ON/OFF）
- 「+ 家庭を追加」ボタンで家庭カードを追加できる
- 家庭・子供は「在籍中」トグルで有効／無効を切り替えられる。無効化された家庭は
  新規イベントの回答・配車対象から除外する（表示上の切り替えのみで、除外ロジック自体は
  データ取得側の責務。本タスクではトグル操作と保存のみを対象とする）

### 保存

- 画面全体の「保存」ボタン、または家庭単位の保存操作で、家庭の追加・編集内容を
  Firestoreへ反映する（保存トリガーの単位は実装時の構成に合わせてよいが、
  04_画面設計.md#10の「保存」ボタン構成を踏襲する）

## 5. 実装範囲外（やらないこと）

以下は対象外。

- 子供の登録・編集UI（T16で実装）
- 集合場所・目的地セクションのUI（T13・T14で実装済み想定）
- Family CRUD処理そのものの実装（T11で実装済みのService層を利用する）
- サンプルデータ投入機能（T17で実装）
- 家庭無効化時の子供連動無効化ロジック（T12・T11側の責務）
- 削除操作（画面設計上は「編集・削除できる」とあるが、本画面（04_画面設計.md#10）は
  在籍中トグルによる論理無効化のみを表現しており、物理削除UIは設計に明記されていないため対象外とする）

これらは別タスクまたは対象外とする。

---

## 6. 受け入れ条件

- マスタ管理画面に「家庭」セクションが表示される
- 登録済みの家庭が一覧表示される
- 「+ 家庭を追加」で新しい家庭カードを追加できる
- 家庭名・コーチ名・通常定員・基本集合場所・在籍中トグルを編集できる
- 基本集合場所は登録済みの集合場所一覧から選択できる
- 在籍中トグルのON/OFFが切り替えられる
- 保存操作でFirestoreへ反映される
