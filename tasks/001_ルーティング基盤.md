# Task001_ルーティング基盤構築

Version: 1.1

---

# 目的

React Routerを導入し、画面遷移の基盤を構築する。

今後の画面追加を容易にするため、
ルーティングのみ実装する。

Firebase・Firestore・認証は実装しない。

---

# 背景

現在はホーム画面のみ存在する。

今後、

- イベント作成
- イベント編集
- 配車画面
- マスタ管理

など画面数が増えるため、
先にルーティング構成を整える。

---

# 完了条件

- React Routerを導入する
- URLによる画面遷移ができる
- ホーム画面が表示される
- イベント作成画面へ遷移できる
- 存在しないURLで404画面が表示される
- npm run build が成功する

---

# 実装内容

## 1. React Router導入

react-router-dom を導入する。

BrowserRouterを使用する。

---

## 2. ルーティング

以下のルートを作成する。

|URL|画面|
|----|----|
|/|ホーム|
|/events/new|イベント作成|
|*|404|

---

## 3. ディレクトリ構成

```
src/

components/

pages/
    HomePage.tsx
    CreateEventPage.tsx
    NotFoundPage.tsx

router/
    AppRouter.tsx

App.tsx
main.tsx
```

---

## 4. App.tsx

App.tsxは画面を持たない。

AppRouterのみ描画する。

---

## 5. HomePage

HomePageを新規作成する。

画面内容はプレースホルダーでよい。

例

```
配車アシスタント

ホーム画面
```

※デザインは最低限でよい。

---

## 6. CreateEventPage

プレースホルダー画面。

```
イベント作成画面
```

と表示するだけでよい。

---

## 7. NotFoundPage

以下を表示する。

```
404 Not Found

ホームへ戻る
```

ホームへ戻るボタンで

```
/
```

へ遷移する。

---

## 8. ナビゲーション

HomePageから

```
イベント作成
```

ボタンを押すと

```
/events/new
```

へ遷移できるようにする。

---

# 実装ルール

- React Router v7系を使用する
- TypeScriptで実装する
- anyは禁止
- 関数コンポーネントを使用する
- React Hooksのルールを守る
- MVPなので過剰な抽象化は禁止
- 分からない仕様は推測せず質問する
